# 运动计划管理 App 验收标准 (Acceptance Criteria)

本文档定义了 App 各核心功能的验收标准，确保开发产出符合用户需求。

## 1. 基础架构与通用要求
- [ ] **免登录**: 打开 App 直接进入首页，无需注册或登录。
- [ ] **数据存储**: 所有数据（动作、计划、记录）均存储在手机本地 SQLite 数据库中，关闭 App 或重启手机后数据不丢失。
- [ ] **重量单位**: 全局统一显示为 `KG`。
- [ ] **UI/UX**: 布局合理，符合移动端操作习惯，支持深色模式（推荐）。

## 2. 动作库模块
- [ ] **动作添加**: 用户可点击添加按钮，输入动作名称（如“杠铃卧推”），保存后实时出现在列表中。
- [ ] **动作列表**: 动作库初始状态应为空（由用户自行录入）。
- [ ] **动作管理**: 支持查看已添加的动作，并能为后续计划选择。

## 3. 计划管理模块
- [ ] **分化选择**: 支持选择“三分化”（胸、背、腿）或“五分化”（胸、背、肩、腿、臂）。
- [ ] **休息设置**: 支持设置循环中的休息天数（如练3休1）。
- [ ] **动作编排**: 
  - 用户能为循环中的每一天设置动作。
  - 必须先选择大类（部位），再从动作库中选择具体动作。
  - 支持设置每个动作的预设“组数”和“次数”。
- [ ] **开始日期**: 用户可以设置/重置计划的开始执行日期。

## 4. 计划日历模块
- [ ] **日历视图**: 根据“开始日期”和“计划配置”自动生成日历，清晰标注每一天的训练内容（如：1号胸、2号背、4号休息）。
- [ ] **打卡功能**:
  - 点击“打卡”展示今日待练动作列表。
  - 允许修改实际完成的组数、次数。
  - **必须支持录入每个动作的重量（KG）**。
  - 点击确定后，数据正确存入运动记录，且日历显示该日已完成。
- [ ] **顺延功能**:
  - 点击“休息”按钮，日历应立即更新，将今日及后续所有计划向后顺延一天。
  - 顺延逻辑应准确，不影响循环的顺序。

## 5. 运动记录模块
- [ ] **列表展示**: 按日期倒序排列（最新的在最前面）。
- [ ] **分页加载**: 记录较多时支持分页加载或滚动加载，保证运行流畅。
- [ ] **日期查询**: 提供日期选择器，能快速跳转到指定日期的运动记录。

## 6. 数据管理 (导入导出)
- [ ] **导出功能**: 一键导出当前 App 所有数据（动作、计划、记录）为本地文件。
- [ ] **导入功能**: 能选择备份文件并还原数据，导入后所有模块数据应恢复至备份时的状态。
